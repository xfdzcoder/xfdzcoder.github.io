不省流版：

详细一点来说：

Java

集合：

  • 线程不安全的：ArrayList、LinkedList、HashSet、TreeSet、HashMap 等
  • 线程安全的：ConcurrentHashMap、CopyOnWriteArrayList 等

Stream流：

  • map：映射
  • filter：过滤
  • collect：收集，一般都是用 Collectors 提供的静态方法

JUC：

  • JMM、volatile、synchronized
  • 线程、锁，虚拟线程（21的新特性，这个还不会
  • CAS、CAS 的 ABA 问题
  • AQS（AbstractQueuedSynchronizer），Doug lea
  • JDK 在他们的基础上提供的工具：ReentrantLock、Lock 等
  • LockSupport、CountDownLatch 等

JVM：

  • 垃圾回收、可达性分析算法
  • JVM 运行时数据区、类加载
  • 还有很多比如锁、线程什么的都和 JUC 重合了

Netty：

  • BIO、NIO、AIO、Reactor模型
  • 计算机网络、七层模型五层模型
  • TCP/IP、UDP、HTTP、HTTP2、RPC

JavaFX：

  • Java 的GUI开发框架，配合 SceneBuilder 还是蛮好用的

Web

H5+C3+JS/TS：

结语

我从不害怕项目遇到bug，因为在经过三年的自学后，我发现：只有不断地发现错误并解决错误才能进步。

想当初刚自学完 Java 基础课程，毫不犹豫的向 韩顺平老师 的《坦克大战》项目发起进攻，那是刚进入大学的第一个寒假，从零开始，先看韩老师的需求，然后自己编码实现，出现 bug 就通过小黄鸭调试法、搜索引擎解决 bug，我用了一整周的时间完成这个项目。那一周一直是在通宵，因为改完一个bug紧接着又来一个bug，甚至有时候一个 bug 能搞一晚上都搞不好。但那终将过去，在那之后我似乎再也不畏惧飘红的报错，有报错说明又可以学到新的东西，这样的进步是极为迅速的。
